浏览器安全检测风险等级分类

检测项目 风险等级 解释
WebRTC 状态 🔴 高风险 可能泄露真实 IP，影响隐私安全。
SNI 加密 (ECH) 🔴 高风险 如果未加密，监听网络的任何人都能看到你访问的网站。
DNS 🔴 高风险 使用传统 DNS（尤其是 ISP 提供的）会暴露你访问的网站，建议使用 DoH 或 DoT 等加密 DNS 服务
远程隔离 (RBI) 🟡 中风险 影响浏览器隔离安全性，恶意代码可能影响本地设备。
TLS 版本 🟡 中风险 旧版 TLS (如 TLS 1.0、1.1) 可能包含安全漏洞。
HTTP 版本 🟡 中风险 HTTP 连接 (非 HTTPS) 可能被监听或篡改数据。
追踪防护 🟢 低风险 通常用于广告追踪，影响隐私但不直接造成安全问题。
匿名 VPN/代理 🟢 低风险 合理使用可靠的 VPN/代理 通常不会影响安全，部分网站会阻止 VPN/代理。
恶意流量过滤 🟢 低风险 一般企业用户才会开启。

常见问题解答

什么是 WebRTC？它会影响我的隐私吗？

WebRTC（Web Real-Time Communication）是一种让浏览器直接进行音视频通话或数据交换的技术。大多数游览器默认开启 WebRTC ，它可能会泄露你的真实 IP 地址，即使你使用了 VPN 或代理。

如何禁用 WebRTC 防止泄露我的 IP？

禁用 WebRTC 需要在浏览器设置中进行调整：

  • Chrome/Edge：Chrome/Edge 的版本较多，非企业版无法完全关闭。仅能通过游览器实验项目、组策略、注册表等方式限制 IP 暴漏，每个版本的方法不同。也可安装游览器插件阻止 IP 泄露，但是，一定要确认扩展插件的安全性。
  • Firefox：在 about:config 里搜索 media.peerconnection.enabled，将其设置为 false。
  • Safari：默认情况下 WebRTC 受到较严格的限制，无需额外设置。
TLS 版本有什么影响？为什么要使用最新版本？

TLS（Transport Layer Security）是一种网络加密协议，影响你的浏览器与网站之间的安全性。较旧的 TLS 1.0 和 TLS 1.1 存在漏洞，可能会被攻击者利用，因此建议使用 TLS 1.2 或 TLS 1.3 以确保安全性。本工具可以检测你浏览器当前支持的 TLS 版本。

检测结果显示了不同的 IP 是什么意思？

如果出现多个不同 ASN 的 IP，意味着你在使用 代理/VPN，甚至可能暴漏了你的真实 IP 地址。

如何使用安全的 DNS？

你可以在浏览器设置中搜索“DNS”并启用加密 DNS（DoH）。这能防止访问记录被 ISP 或第三方监听。

推荐使用支持加密 DNS 的服务商，例如：

  • Cloudflare 1.1.1.1
  • Google DNS
  • AdGuard DNS

启用后，你的 DNS 请求将通过加密通道发送，提升匿名性和安全性。

SNI（Server Name Indication）是什么？它为什么重要？

SNI 是 TLS 连接中的一个扩展，它允许服务器在加密过程中识别访问的域名。普通 SNI 信息是明文的，可被监视。启用 加密 SNI（ECH） 可以防止这种信息泄露，本工具可以检测你的浏览器是否支持加密 SNI。

如何提高游览器安全？

适合懒人、小白用户的简单方法：

  • 使用隐私浏览器（如 Brave 或 Firefox）以减少指纹暴露。
  • 禁用 WebGL 渲染，降低设备指纹识别风险。
  • 启用 “抗指纹” 保护（如 Firefox 的 privacy.resistFingerprinting 选项）。
  • 使用真正的 VPN 。绝大多数真 VPN 会自动解决 WebRTC、DNS、SNI 等的安全问题。
  • 谨慎安装插件，插件会获取游览器管理员权限，是隐私泄露的重灾区。
为什么我的浏览器安全检测显示“未启用”某些防护？

可能的原因包括：

  • 你的浏览器不支持某些安全功能，例如 TLS 1.3 或 加密 SNI。
  • 你的网络提供商屏蔽了某些隐私保护技术，例如 VPN 或加密 DNS。
  • 你的浏览器未正确配置安全设置。

建议查看你的浏览器隐私设置，并使用适当的安全工具进行调整。
