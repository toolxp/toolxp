反伪装检测

一键检测是否使用了 代理/VPN 等匿名手段

通过链路测试、地理位置延迟、指纹真实性、POP 最优入口等方式，检测网络连接是否真实
助你识别 VPN、代理或伪装行为。


检测项目说明

检测项目说明
时区不一致浏览器与 IP 时区不符，此项误伤率高，伪造无门槛，参考价值极低。
爬虫/滥用/数据中心 IP通过 IP 数据库 查询是否被标记为爬虫IP、滥用IP、数据中心 IP，此项结果从技术层无法直接关联是否为 代理/VPN 或其他伪装，仅在行为层做辅助判断，参考价值极低。
代理/VPN 标记各个 IP 风险数据库 的判定结果可能存在差异，即使被标记的 IP 也有误判的可能，虽然有一定的参考价值，仍需结合其他检测指标综合判断，尽可能减免误伤。
链路延迟测量中继/隧道行为信号伪造难度中等，网络波动影响可能会误伤，具有一定的参考价值。
WebRTC 泄露真实 IP 与公网 IP 不一致是强信号，参考价值较高。但双栈网络、运营商 NAT+多出口、ISP 多出口、设备/路由级干扰等因素可能会导致误伤。（注：全球 WebRTC 检测都不稳定，这种全球间歇性不稳定与检测方法和 STUN 没有必然关系，简单理解为：不可控。若本工具此项检测失效，你可以使用其他任何工具进行检测验证——你会发现它们也全部失效。）
操作系统真实性 通过指纹推测真实操作系统，若协议层与应用层信息不一致，是伪装的强信号。伪造难度较高，误伤率低，参考价值高。（注：1、TCP/TLS 指纹检测对部分移动设备无效，尤其是 Android 手机；2、容器、虚拟机、自定义系统环境及部分安全软件可能会导致误判。）
地理位置延迟检测地理位置延迟异常是强信号，伪造需要真实跨境链路或模拟物理距离的 RTT，难度很高，误伤率较低，有可能受链路波动影响，参考价值高。
POP 最优入口检测CDN POP 入口异常是极强信号，伪造需要专业技术+持续资金投入，难度极高，误伤率极低，如果此项检测到异常，参考价值非常高。（注：极少数教育网/企业网，跨境线路，IPv6-only 可能有例外。）
TOR 标记TOR 是公开匿名网络，几乎不存在误伤，参考价值极高。

已删除检测项说明

检测项删除理由
HTTP Headers Test能被轻易伪装，独立检测几乎无意义，已合并到操作系统真实性检测中进行综合判断。
网络流量模式检测易被绕过，误判率高，检测精度有限。
IP 多出口检测仅对少数特定国家/地区有效（如大陆），局限性大，不通用。
浏览器与 IP 语言一致性几乎没有网站将此项作为风控指标，无参考价值。

常见问题解答

工具是否会泄露我的隐私？

不会。本工具不记录、不分享、不存储、不上传你的任何个人信息。所有检测结果均在本地浏览器完成并仅在你当前页面中展示，你可以放心使用。

使用“指纹浏览器 + 住宅 IP”也能被检测到 代理/VPN 吗？

是的，使用“指纹浏览器 + 住宅 IP”依然可能被本工具检测到伪装。
这是因为一些高权重关键指标，如 TCP/TLS 指纹、网络链路特征以及 CDN POP 调度等，属于指纹浏览器无法伪造的底层信号。

“链路延迟测量”是什么意思？

测量检测节点与连接来源之间的网络延迟，判断是否存在中继或隧道行为。如果 RTT 明显偏高，可能是因为网络经过多个中转节点，属于潜在伪装信号。

“操作系统真实性”检测是如何工作的？

首先通过分析指纹（如 TTL、TCP 选项顺序等），推测访问者所使用设备的真实操作系统类型。然后将其与浏览器声明的操作系统（如 User-Agent 中的 Windows、Android 等）进行比对。如果两者不一致，是判断为伪装的强信号。

“地理位置延迟检测”是怎么判断的？

测量连接来源和服务器之间的连接延迟，如果延迟表现与地理位置的标准延迟加上容错阈值严重不符，说明可能在使用远程节点或伪装链路。

“POP 最优入口检测”是做什么的？

这是检测连接来源被 CDN 调度到哪个入口节点。如果与真实网络不一致，属于极强伪装信号。
